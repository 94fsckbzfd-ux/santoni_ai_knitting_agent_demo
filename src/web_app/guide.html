适用范围

  • Designer：通过自然语言描述设计目标，进行设计意图澄清、产品开发 brief 探索、模拟针织/SWS 方向建议。
  • Customer Equipment Engineer：描述设备问题，Agent 先尝试在线协助，再判断是否需要派工或生成服务工单。
  • Service Case Review：查看当前 mock service case 的审核状态、在线处理步骤、安全提醒和派工触发条件。
  • 锁机激活密码：在受控流程下测试机器锁定后的激活密码生成场景，可使用 mock 或实验性真实平台连接。

不适用范围

  • 当前版本不连接真实 SWS、APS、DPP、正式工单系统或客户设备数据。
  • 图片上传目前只保留接口和 mock 理解结果，不做真实视觉识别。
  • Service case 中由 Excel 提炼的案例仍需要 Santoni 服务团队审核，不能直接作为正式客户维修指令。
  • 涉及安全风险、拆机、电气维修、停机损失或备件更换的判断，需要人工服务人员确认。

使用方法

  1. 打开 Web Demo 首页，直接用自然语言说明你的身份或需求，不需要手动选择角色。
  2. 业务用户使用默认首页；开发和测试同事可以进入 /developer.html 查看完整调试信息。
  3. Designer 测试可以输入产品类型、目标用户、使用场景、风格、功能、材料、成本或交期限制。
  4. Service 测试可以输入机型、序列号、故障现象、报警信息、生产影响、现场已尝试步骤和图片证据。
  5. 需要重新开始一轮测试时，点击首页的 Clear Memory。
  6. 测试结束后点击 Export Test Log，把 JSON 日志发给开发侧分析对话过程。
  7. 测试锁机激活密码或解锁机器真实平台时，用户页会在识别到任务后弹出平台账号密码输入框。

当前功能

  • 主 Agent 根据自然语言识别 Designer 或 Service 场景。
  • Designer Agent 会先追问设计意图，避免过早生成方案。
  • Service Agent 会收集工单必要信息，并优先尝试安全的在线协助。
  • Service Case Online Assist Mock 可以匹配结构化服务案例并给出下一步检查建议。
  • Service Case Library 页面可以查看案例状态和 Excel 自动导入后的待审核 draft cases。
  • 支持中文和英文测试输入。
  • 支持导出测试日志，日志不包含上传图片内容和平台密码。

Excel 导入

  • 当前版本支持通过本地脚本把服务 Excel 转换成待审核 draft service cases。
  • 默认输出文件是 src/mock_data/service_cases.draft_import.json。
  • 导入结果会显示在 Service Case Library，状态为 draft_needs_review。
  • draft cases 不会自动进入客户回答逻辑，必须先在 Service Case Library 标记为 approved。
  • needs_changes 和 internal_only 会保留在审核页，但不会进入客户回答逻辑。

测试建议

  • 分批输入信息，观察 Agent 是否能保留上下文。
  • 故意缺少关键字段，观察 Agent 是否追问而不是直接生成结果。
  • 测试 Designer 时重点观察是否理解设计意图、功能需求、用户场景和限制条件。
  • 测试 Service 时重点观察是否先在线协助、是否正确判断派工条件、是否避免过度承诺。
  • 测试 Service case 时，可以参考 Service Case Library 中的关键词和机型进行对话。

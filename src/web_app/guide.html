Scope

  • Designer: describe design goals in natural language to clarify intent, explore product-development briefs, and prepare future SWS direction.
  • Customer Equipment Engineer: describe equipment issues so the Agent can try online assistance before deciding whether dispatch or a service ticket is needed.
  • Service Manager Console: review and edit service case status, online steps, safety warnings, dispatch triggers, and future ticket payloads.
  • Lock-machine activation: test controlled activation-password generation after machine lock, using mock or experimental real-platform connection.

Out of Scope

  • The current version does not connect to real SWS, APS, DPP, formal ticket systems, or customer equipment data.
  • Image upload currently keeps interface and mock-understanding results only; it does not perform real visual recognition.
  • Service cases extracted from Excel still need Santoni service-team review before they become formal customer-facing guidance.
  • Safety risk, machine disassembly, electrical repair, downtime loss, or spare-part replacement decisions require human service confirmation.

How to Use

  1. Open the web demo home page and describe your identity or need in natural language. You do not need to manually select a role.
  2. Business users use the default home page; development and testing users can open /developer.html for full debug information.
  3. Designer tests can include product type, target user, use case, style, function, material, cost, or delivery constraints.
  4. Service tests can include machine model, serial number, symptom, alarm, production impact, attempted steps, and image evidence.
  5. When you need a fresh test round, click Clear Memory on the home page or developer page.
  6. After testing, export the JSON test log for development-side analysis.
  7. For real-platform lock-machine activation tests, the customer page prompts for platform credentials only after detecting the task.

Current Features

  • Main Agent identifies Designer or Service scenarios from natural language.
  • Designer Agent asks follow-up questions before generating a plan too early.
  • Service Agent collects required ticket information and prioritizes safe online assistance.
  • Service Case Online Assist Mock matches structured service cases and suggests next checks.
  • Service Manager Console supports case status review, case knowledge editing, diff view, and future CRM / ticket payload preview.
  • Production Operations Console monitors order, ERP, APS, IOT, production/service escalation, and garment output with KPI and evidence logs.
  • All web pages provide a Chinese / English language switch.
  • Test logs can be exported and do not include uploaded image content or platform passwords.

Excel Import

  • The current version supports converting local service Excel files into draft service cases for review.
  • The default output file is src/mock_data/service_cases.draft_import.json.
  • Imported cases appear in Service Case Library with draft_needs_review status.
  • Draft cases do not enter customer-facing matching until Service Manager Console marks them approved.
  • needs_changes and internal_only cases remain in review pages but do not enter customer-facing response logic.

Testing Suggestions

  • Enter information in batches and observe whether the Agent keeps context.
  • Intentionally omit key fields and check whether the Agent asks follow-up questions instead of inventing results.
  • When testing Designer, focus on whether it understands intent, functional needs, user scenarios, and constraints.
  • When testing Service, focus on online assistance, dispatch conditions, and avoiding over-promising.
  • When testing service cases, use keywords and machine models from Service Case Library as references.

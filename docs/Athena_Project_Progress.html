说明与计算规则
初始进度是根据当前 Athena 架构文档建立的管理基线，不代表最终承诺。建议每次架构会、研发会或客户试点复盘后更新一次，并导出 JSON 作为阶段记录。
PMO First：先确认 PMO Agent / Project Intelligence Agent，再推进各模块细节。它负责把 Belotti 的战略判断拆成 roadmap、Owner、MVP 范围、验收 checklist、评审节奏、证据来源和决策机制。
延期状态由页面自动计算：目标日期已过且未完成为 Delayed；Blocked 或临近目标日期但进度偏低为 At Risk；已完成为 Done；未设置目标日期或未来模块为 TBD。
项目进度由页面自动计算：MVP 状态占 50%，项目状态占 20%，商业化状态占 30%。MVP：TBD/No=0，Partial=50，Yes=100；状态：Future=0，Planning/Blocked=20，In Progress=50，Review=80，Done=100；商业化：TBD=0，Not sellable=20，Internal Enablement=40，Demo=60，Pilot=80，Sellable=100。

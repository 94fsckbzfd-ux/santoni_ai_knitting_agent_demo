AI Knitting Agent 商业化落地方案讨论

本材料用于内部讨论 AI Knitting Agent 从免费入口、3 个月 MVP、付费模块、试点客户到长期商业化路径的落地方案。

讨论目标 形成 MVP 测试与试点共识
时间窗口 3 个月后推出 MVP
首批客户 现有大客户与关系好的 OEM/ODM 工厂
核心商业模式 SaaS + 模块包 + 积分

一、已形成的基本判断

AI Knitting Agent 应被定位为面向针织产业链的智能生产与运营 Agent，而不是单一 AI 设计工具。它的长期价值在于连接设计、打样、排产、设备维护、工单、机器数据与订单 ROI，让工厂和品牌方在效率、确定性和经营决策上获得可衡量提升。

AI 设计打样 MyKnit / SWS Factory Intelligence Package APS 排单优化 设备 Knowledge Base 订单 ROI 概念验证 Advanced Maintenance Package

二、目标客户与关键角色

OEM / ODM 工厂

第一阶段商业化重点客户。尤其适合 50 台以上机器、有打样、排产、插单和设备管理复杂度的工厂。

  • 老板 / 管理层：关注订单 ROI、利润、开机率、交付确定性。
  • IT 部门：负责项目管理、系统落地、账号和积分分配。
  • 打样师：减少与设计师沟通时间，快速出样。
  • 排单人员：提升排单、插单效率和机器利用率。
  • 挡车工 / 车间技术工：快速查询设备问题和操作建议。
  • 机修团队：处理工单、预警机器和维护建议。

品牌方

品牌方不是第一优先级，但可以作为设计打样能力的外部客户与生态入口。

  • 设计师团队：快速生成服装原型，查看仿真渲染效果。
  • 供应链团队：更清楚地向工厂表达需求，减少样衣沟通轮次。
  • 采购 / 管理层：如果愿意采购，可放宽使用边界。

三、3 个月 MVP 建议范围

第 1 阶段：免费入口

  • Web 版无需登录即可使用。
  • 接入设备相关 Knowledge Base。
  • 支持现场技术人员快速问答。
  • 支持基础问题诊断与工单创建。
  • MyKnit 作为自然入口，引导客户创建账号。

第 2 阶段：设计打样 MVP

  • 基于已有 SWS 和升级中的 MyKnit。
  • 支持设计原型生成。
  • 支持款式说明、打样沟通材料和基础工艺建议。
  • 逐步加入仿真与渲染图生成。
  • 主要面向 OEM/ODM 工厂，品牌方可销售。

第 3 阶段：工厂智能包试点

  • APS 不独立销售，纳入 Factory Intelligence Package。
  • 验证排单优化、插单响应和机器资源匹配。
  • 订单 ROI 先作为概念验证，不在 MVP 中过度承诺。
  • 面向 50 台以上机器的工厂优先试点。

四、产品包与收费逻辑

层级 目标客户 核心能力 收费逻辑
Free AI Knitting Agent 所有客户和潜在客户 设备知识库问答、基础故障辅助、工单创建、基础 MyKnit 入口 免费；不限制普通问答次数；引导创建账号

五、订阅版本与积分机制

基础版

  • 适合轻量试用和小团队。
  • 包含基础账号和机器数量。
  • 包含免费 Agent 增强能力和基础 MyKnit 功能。

进阶版

  • 适合有明确打样和协作需求的工厂。
  • 提供更多账号、机器额度和团队功能。
  • 包含更完整的 AI 设计打样能力。

专业版

  • 适合 50 台以上机器的中大型工厂。
  • 包含 Factory Intelligence Package。
  • 未来加入订单 ROI、老板驾驶舱和高级报表。

积分机制建议

积分同时绑定公司账户和个人账户。公司账户拥有总积分池，客户方管理员将积分分配给设计师、打样师、排单人员、IT、机修团队等个人账户。这样可以让复杂功能用量由客户内部管理，减少 Santoni 与客户使用部门之间围绕额度不足产生的直接矛盾。

建议消耗积分的复杂交互 原因
完整服装设计方案、批量款式方案、仿真渲染 计算成本高，且直接影响客户商业效率。
SWS / MyKnit 高级工艺生成 属于高价值专业能力，应与免费问答区分。
APS 排产优化、插单影响模拟 涉及复杂算法和工厂生产决策。
订单 ROI 分析、老板报表 直接对应管理层付费意愿。
远程诊断报告、机器健康分析、月度设备运行报告 适合纳入 Advanced Maintenance Package 或高阶积分场景。

六、订单 ROI 算法建议方向

订单 ROI 目前应作为产品力方向和老板层销售故事，不建议在 3 个月 MVP 中承诺完整算法。建议先通过内部讨论和 OEM 工厂老板访谈，确认真正影响接单决策的数据口径。

ROI Lite 数据需求

  • 订单金额、数量、单价和交期。
  • 用纱类型、用纱量和纱线单价。
  • 预计生产时间、机器类型和机器数量。
  • 人工成本、打样成本和编程成本。
  • 当前排产队列和机器空闲情况。
  • 预计毛利、交付风险和是否值得接单。

ROI Pro 数据需求

  • 机器实际 OEE / 开机率。
  • 换款、换纱、调机时间。
  • 历史返工率、次品率和质量风险。
  • 停机概率、机修风险和备件风险。
  • 客户付款周期、复购可能性和战略价值。
  • 接单与不接单的产能机会成本。

七、试点销售路径

首批试点客户

  • 现有 Santoni 大客户。
  • 关系好的 OEM/ODM 工厂。
  • 优先选择 50 台以上机器、有真实排产和打样痛点的客户。
  • 品牌方如有明确采购意愿，可作为设计打样方向试点。

销售价值故事

  • 短期主打设计打样提效。
  • 中期主打 Factory Intelligence Package。
  • 长期主打老板能看清订单 ROI。
  • 未来可考虑面向竞争对手设备较多的工厂提供 Smart Machine 改造服务。
老板层最容易感知的指标 建议表达方式
打样周期缩短时间 从“几天出样”变成“更快形成可讨论方案和可执行工艺方向”。
开机率提升 减少等待、空转、排产冲突和异常停机带来的产能损失。
插单响应速度 让插单影响可见，快速判断是否值得调整现有排产。
每月节约人工 减少重复沟通、人工排单、人工查资料和手工汇总报表。
订单利润更清楚 把经验判断转化成可记录、可比较、可复盘的数据决策。

八、建议新增讨论点

1. MVP 的成功标准

3 个月后的测试不能只看功能是否上线，还要定义客户是否愿意继续使用、愿意创建账号、愿意提供数据和愿意付费试点。

2. 数据授权与边界

工厂数据涉及订单、成本、排产和设备状态，需要提前定义哪些数据必须采集、哪些只做本地化处理、哪些可匿名化用于算法优化。

3. MyKnit 与 Agent 的入口关系

需要明确客户第一眼看到的是 MyKnit、AI Knitting Agent，还是二者融合后的统一入口。

4. 内部责任分工

研发、应用、售后、销售、IT 和产品管理需要明确谁负责知识库、谁负责客户试点、谁负责数据接口、谁负责商业报价。

5. 与 Smart Machine 的节奏配合

Advanced Maintenance Package 很重要，但不应拖慢 3 个月 MVP。建议与传感器选型和数据采集系统并行推进。

6. 竞争对手设备接入策略

未来如果向竞争对手设备较多的工厂提供改造服务，需要讨论技术标准、服务边界、商业风险和销售话术。

九、内部会议建议 Agenda

  1. 确认产品定位
    AI Knitting Agent 是免费工具、设计工具、设备服务，还是工厂智能化平台？建议统一为“针织产业链智能生产与运营 Agent”。
  2. 确认 3 个月 MVP 范围
    是否同意 MVP 聚焦免费 Web Agent、设备 Knowledge Base、基础工单、MyKnit 入口和 AI 设计打样，而不把 Advanced Maintenance 作为首要交付。
  3. 确认免费版与积分边界
    普通问答是否永久免费？哪些交互定义为复杂交互？积分是否按公司账户统一购买并由管理员分配？
  4. 确认套餐结构
    基础版、进阶版、专业版分别包含哪些功能、账号数、机器数和积分额度？超出后如何收费？
  5. 确认试点客户名单
    从现有 Santoni 大客户和关系好的 OEM/ODM 工厂中筛选 3 到 5 家试点客户，明确联系人、场景和可开放数据。
  6. 确认 ROI 概念验证方式
    是否先访谈 OEM 工厂老板，收集接单决策、成本结构、排产风险和利润判断逻辑，再定义 ROI Lite。
  7. 确认内部资源与负责人
    明确产品、研发、销售、售后、应用工程师和 IT 的责任边界，以及每两周的里程碑检查机制。
